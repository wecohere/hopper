    Contributing to Hopper

    Back home

    Hopper is written in javascript, and targets the nodejs runtime. If you’d like to contribute to hopper development, there’s a few things you should know:

    • We have a Code of Conduct.
    • We use Github Flow for contributions.
    • We have a Work in Progress board.
    • We love bug reports, documentation improvements, and/or request features being added to our issue tracker.

    Creating a Development Environment for Hopper

    1. Ensure you’ve installed node.
    2. Fork and clone the hopper repository>
    3. Open the directory you cloned the repository into with your favorite text editor.
    4. Run npm install to install the dependencies.
    5. Run make test docs to run the tests and build the docs!
    6. Run ./libexec/hop to run your local version of hopper.

    Submitting a Patch

    Once you’ve set up your development environment, you’re ready to tackle adding a feature, improving the documentation or fixing a bug.

    1. Create a branch that describes the feature you’re building/bug you’re fixing
    2. Type Type Type!
    3. Run make test docs frequently! (I recommend using wach to re-run the tests and docs on every file save)
    4. Push your branch to Github.
    5. Submit a pull request
    6. Adapt your code based on feedback from peers and project contributors
    7. Watch your patch get merged
    8. Revel in the adoration of your peers and professional associates!
